# Coverage Report

This directory is the target for HTML coverage reports generated by `ninja coverage-html`.

## Generating a Coverage Report

Coverage requires GCC with `gcov` and `lcov` installed (Linux only — IOCP coverage on Windows requires a separate LLVM approach).

```bash
# 1. Configure with coverage instrumentation
meson setup builddir-cov -Db_coverage=true -Dbuildtype=debug

# 2. Build and run tests (this populates .gcda files)
meson test -C builddir-cov

# 3. Generate the HTML report
ninja -C builddir-cov coverage-html

# 4. Open the report in a browser
xdg-open builddir-cov/meson-logs/coveragereport/index.html
# or on macOS:
open builddir-cov/meson-logs/coveragereport/index.html
```

The HTML report is written to `builddir-cov/meson-logs/coveragereport/`.

## CI Usage

```bash
meson setup builddir-cov -Db_coverage=true -Dbuildtype=debug
meson test -C builddir-cov --no-rebuild
ninja -C builddir-cov coverage-xml   # produces coverage.xml for CI upload
```

The XML report (`builddir-cov/meson-logs/coverage.xml`) is compatible with Cobertura / GitLab coverage parsing and GitHub Actions coverage reporters.

## Coverage Targets (meson)

| Command | Output |
|---|---|
| `ninja -C builddir-cov coverage-html` | Interactive HTML report |
| `ninja -C builddir-cov coverage-xml` | Cobertura XML (CI upload) |
| `ninja -C builddir-cov coverage-text` | ASCII summary to stdout |

## Notes

- `builddir-cov/` is in `.gitignore` — coverage build artifacts are never committed
- Coverage of `platform/iocp.c` requires running on Windows; use `platform/epoll.c` coverage on Linux CI
- Minimum coverage target: **80%** on `src/` (config, router, balancer, tunnel, log, utils)
