proxy-epoll-iocp

High-performance Layer 7 reverse proxy in C — async I/O via epoll (Linux) and IOCP (Windows).

Architecture

The proxy listens on one or more ports. Each incoming connection is dispatched by an event loop (epoll on Linux, IOCP on Windows) without blocking threads. Routing decisions are made at Layer 7 using the HTTP Host header, and upstream traffic is distributed across backend servers via round-robin load balancing.

Client A ──┐ Client B ──┤──▶ [ Listener : port 8080 ] Client C ──┘ │ ▼ [ Event Loop (epoll/IOCP) ] │ [ Router — domain match ] / \ domain-a.example.com domain-b.example.com │ │ [ Balancer RR ] [ Balancer RR ] / \ / \ backend:3001 :3002 backend:4001 :4002

Key Features

  • Layer 7 (HTTP) reverse proxy
  • Domain-based routing (Host header)
  • Multiple independent listen ports
  • Round-robin load balancing per upstream
  • Dynamic configuration reload (no restart)
  • Async I/O — epoll on Linux
  • Async I/O — IOCP on Windows
  • TOML-based configuration (proxy.toml)
  • Built and tested with Meson
  • Unit tests via Unity framework

Quick Start

# Clone the repository
git clone https://github.com/Jorgeaapaz/MISEIA_1-6-10-revproxy-c.git
cd MISEIA_1-6-10-revproxy-c

# Configure and build (Linux)
meson setup build
meson compile -C build

# Run unit tests
meson test -C build

# Start the proxy (uses proxy.toml by default)
./build/proxy proxy.toml

# Reload configuration without restarting
kill -HUP $(pgrep proxy)
